1. Brand Overview & Mission


Brand Name: Canadian Optimizer
URL: CanadianOptimizer.com
Owner: Andrew Carrothers
Contact: andrew@canadianoptimizer.com

Mission Statement
Canadian Optimizer provides Canadians with actionable strategies and tips to optimize their financial life with the goal of maximizing wealth. The site focuses on specific optimization strategies around each financial topic, going beyond basic explainers to deliver tactics readers can implement immediately.

Differentiation
The core differentiator is the optimization angle. Unlike general Canadian personal finance sites that explain concepts, Canadian Optimizer assumes readers understand the basics and focuses on extracting maximum value. Every piece of content should answer the question: “How do I optimize this specific aspect of my finances?”

Target Audience
	•	Canadians seeking to maximize their financial outcomes
	•	Readers who understand basic financial concepts and want advanced strategies
	•	Individuals looking for specific, actionable optimization tactics
	•	Canadians navigating tax optimization, investment strategy, and wealth building

8. Content Pipeline & AI Workflow


Pipeline Overview
Blog posts are generated using AI through a content pipeline, with mandatory human review before publication. No post is published automatically — all posts enter “Draft” status and require manual approval.

Pipeline Process

Step 1: Topic Input
A topic is submitted to the content pipeline for blog post creation.

Step 2: Research & Validation
	•	Grok API or Gemini API performs research on the topic
	•	Research is validated for accuracy and current information
	•	All specific numbers, CRA references, and tax rules are verified against official sources
	•	Provincial differences are identified where applicable

Step 3: Content Generation
The AI generates a blog post based on the validated research. The post must meet the following requirements:

	•	Hook: Opening includes an interesting data point or provocative statement to capture attention and increase retention
	•	Structure: Organized into logical sections that take the reader through a coherent storyline and flow
	•	Tone: Engaging and educational, focused on specific optimization strategies
	•	Formatting: Uses headings, subheadings, and visual separation between content sections
	•	Value: Includes specific data, insights, strategies, examples, and use cases
	•	Media: Includes relevant data tables, charts, diagrams, and images
	•	SEO: Optimized for search engines with proper keyword usage
	•	Length: Minimum 1,000 words

Step 4: Human Review (Mandatory)
All AI-generated posts enter “Draft” status. The owner reviews each post before publication using the following checklist:

	•	Verify all specific numbers and statistics against primary sources
	•	Check CRA references against actual CRA pages
	•	Confirm all strategies apply for the current tax year
	•	Verify provincial regulations where mentioned
	•	Review tone and ensure optimization angle is clear
	•	Check that content adds genuine value beyond what free competitors offer
	•	Verify all images have descriptive alt text

Step 5: Publication
Once approved, the post is published to the website under the correct content category. The pipeline automatically assigns the post to the appropriate category page.

9. SEO & LLM Optimization


Since this site is built for the AI era, the invisible design is as important as the visual one.

Semantic HTML
	•	Wrap main content in <main> element
	•	Use <article> for blog posts
	•	Use <aside> for sidebars
	•	Use proper heading hierarchy (H1 → H2 → H3)

Alt Text Strategy
Every image must have descriptive alt text. Examples:
	•	“Ebook cover for The Canadian Mortgage Strategy Guide”
	•	“Chart showing TFSA contribution room growth from 2009 to 2026”
	•	“Comparison table of top Canadian high-interest savings accounts”

Breadcrumbs
Implement breadcrumb navigation to help LLMs and search engines understand site structure. Example: Home > Investing > TFSA Optimization Strategies

Schema Markup (JSON-LD)
	•	“Article” schema for all blog posts (includes author, date, headline)
	•	“Product” schema for ebook pages (includes pricing and availability)
	•	“BreadcrumbList” schema for navigation paths
	•	“Organization” schema for the site

YMYL & E-E-A-T Considerations
As a financial content site, Canadian Optimizer falls under Google’s YMYL (Your Money, Your Life) guidelines. This means E-E-A-T signals are critical:
	•	Experience: About page must demonstrate personal experience with financial optimization
	•	Expertise: Author bio with real credentials on every blog post
	•	Authoritativeness: Cite official sources (CRA, Bank of Canada, OSFI) in content
	•	Trustworthiness: Clear disclaimers, transparent affiliate disclosures, accurate information

10. Monetization Strategy


Revenue Streams

1. Google AdSense
Display advertising integrated throughout the site. Ad placements should not compromise user experience or content readability.

2. Affiliate Links
Affiliate partnerships with Canadian financial products and services. All affiliate relationships must be disclosed clearly per Canadian advertising standards and FTC guidelines.

3. Digital Ebook Downloads
Paid ebooks sold directly through the website via Stripe checkout. Pricing in CAD. Each ebook has a dedicated product landing page with purchase flow.

Disclosure Requirements
	•	All pages with affiliate links must include a visible affiliate disclosure
	•	Ebook product pages must clearly display pricing in CAD before checkout
	•	A general disclaimer page must be accessible from the footer

11. Email & Newsletter Strategy


Lead Magnet
A free ebook offered in exchange for email signup. The free ebook should be genuinely useful and reference-worthy (e.g., “The Canadian Tax Optimization Checklist” or “5-Day Optimization Starter Guide”). This builds trust and drives newsletter engagement more effectively than a generic overview.

Email Capture Placements
	•	Sticky header “Join Free” button (always visible)
	•	Dedicated newsletter section on the homepage (red background CTA)
	•	End-of-blog-post signup prompt
	•	Exit-intent popup (optional, high conversion)
	•	Sidebar widget on blog posts (if sidebar is implemented)

Newsletter Content
Regular newsletter delivering optimization tips directly to subscribers. Content should drive traffic back to the website and promote new blog posts and ebook releases.

Navigation

13. Version Control & Deployment


Git Repository
Repository URL: https://github.com/drewbotcarrothers/canadianoptimizer
Hosting Provider: Hostinger.com
Deployment Method: GitHub Actions → FTP to Hostinger

Branch Strategy
	•	main — Production branch. Merges to main trigger automatic deployment to Hostinger via FTP.
	•	develop — Development branch for staging and testing changes before merging to main.
	•	feature/* — Feature branches for individual changes (e.g., feature/ebook-checkout, feature/new-category-page).

GitHub Actions Deployment Workflow
A GitHub Actions workflow is configured at .github/workflows/deploy.yml that automatically builds and deploys the Next.js site to Hostinger via FTP whenever code is pushed to the main branch.

Workflow Steps
	•	Trigger: Push to main branch
	•	Checkout repository code
	•	Set up Node.js environment
	•	Install dependencies (npm ci)
	•	Build the Next.js project (npm run build / next export for static)
	•	Deploy built output to Hostinger via FTP using lftp or ftp-deploy action

Required GitHub Secrets
The following secrets must be configured in the GitHub repository settings under Settings → Secrets and variables → Actions:

Secret Name
Description
FTP_SERVER
Hostinger FTP server hostname (e.g., ftp.canadianoptimizer.com or provided by Hostinger)
FTP_USERNAME
FTP username for the Hostinger hosting account
FTP_PASSWORD
FTP password for the Hostinger hosting account

Workflow File Reference
The deploy.yml workflow file should be placed at .github/workflows/deploy.yml in the repository. It uses the SamKirkland/FTP-Deploy-Action or equivalent to sync the built output directory to the Hostinger public_html directory via FTP/FTPS.

Deployment Checklist
	•	GitHub repository created and accessible
	•	FTP credentials obtained from Hostinger control panel
	•	GitHub Secrets configured (FTP_SERVER, FTP_USERNAME, FTP_PASSWORD)
	•	deploy.yml workflow file committed to repository
	•	Test deployment triggered and verified on live site
	•	Confirm Next.js build output structure matches Hostinger’s expected directory (public_html)
	•	Verify HTTPS/SSL certificate is active on Hostinger for canadianoptimizer.com
